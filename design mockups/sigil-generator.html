Master sigil · generative pattern

The sigil is the master's portrait when no photo exists. It's a deterministic generative pattern derived from the master's ID — the same input always renders the same sigil, so a master's identity stays visually consistent across cards, modals, OG images, and business cards.

The algorithm

Generation rules

  1. Grid: N × N cells — N = 4 or N = 3
  2. Shapes: square · circle · triangle · bar · empty
  3. Colors: ink or terra (terra is the accent)
  4. Fill rate: 6–10 of 16 cells (4×4), 4–6 of 9 cells (3×3)
  5. Seed: stable master identifier — rendering is deterministic

The seed string (master ID, e.g. m12) is hashed with FNV-1a to a 32-bit integer, then fed into a Mulberry32 PRNG. That gives a reproducible stream of uniform values, used in this order:

1 pick fill count from the valid range   2 partial Fisher–Yates to choose which cells fill   3 per filled cell, pick shape & rotation   4 pick which of the filled cells get the terra accent (≈20–30%, at least one).

Squares fill the cell edge-to-edge so adjacent same-color squares read as one chunky block — the brutalist look. Circles, triangles, and bars sit inside a small inset. Triangles and bars rotate in 90° steps so two cells with the same shape rarely look identical.

The 3×3 variant trades variation for legibility — fewer cells mean each one is bigger, so the sigil reads as an icon at small sizes (avatar in a search row, favicon, OG thumbnail). The 4×4 is the canonical specimen: more visual complexity, better for hero use (modal, OG share, business card).
